# Revenue Growth Framework

## 売上分解モデル

### SaaS/サブスク型
```
MRR = 新規MRR + 拡張MRR - チャーンMRR
    = (新規顧客数 × ARPU) + (既存アップセル) - (解約顧客数 × ARPU)
```

### EC/物販型
```
売上 = 訪問数 × CVR × 客単価 × 購入頻度
```

### サービス/受託型
```
売上 = 案件数 × 平均単価 × 稼働率
```

## 成長レバー特定（佐藤裕介思考）

### Step 1: 現状の分解
各変数の現状値を把握し、ボトルネックを特定

### Step 2: レバレッジ判定
| レバー | 改善難易度 | PLインパクト | 優先度 |
|---|---|---|---|
| 新規獲得 | 高（CACが上昇傾向） | 中 | B |
| CVR改善 | 中（UX改善で対応可） | 高 | A |
| 単価UP | 低（価格改定で対応可） | 高 | A |
| チャーン削減 | 中（CS強化必要） | 高 | A |

### Step 3: 施策のPLシミュレーション
各施策の投資額×期待リターン×回収期間を試算

## 小野寺チェックポイント
- フロー（広告投資）に偏重していないか
- ストック（ブランド/SEO/CRM）への投資比率は適切か
- 1stPartyデータで効果測定できる設計になっているか

## アンチパターン
- 「とりあえず広告費を増やす」→ CACの限界利益を超えていないか確認
- 「新機能を追加すれば売れる」→ 既存機能の価値伝達が先（佐藤裕介: プロダクトバリューを疑え）
- 「競合が安いから値下げ」→ 価格以外の差別化要因を探る
