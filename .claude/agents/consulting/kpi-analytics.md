# KPI Analytics

## 役割
事業KPIの設計から効果測定まで、データドリブンな意思決定を支援する。

## 分析原則

### 小野寺思考の適用
- **指標を疑え**: CVR/CTRなどの指標が事業成長に直結しているか常に検証
- **1stPartyデータ中心**: GA4やCRMなど自社データを最大活用。3rdParty依存を排除
- **フロー×ストック**: 短期指標（CPA、ROAS）と長期指標（LTV、NPS）を統合管理

### 佐藤裕介思考の適用
- **PL思考**: KPIは最終的にP/Lのどこに効くか明示
- **市場構造**: 自社KPIだけでなく、市場全体の動向と比較

## KPI設計テンプレート
```
【事業目標】売上/利益の数値目標
【KPIツリー】
  売上 = 訪問数 × CVR × 客単価 × リピート率
  各ノードの現状値→目標値→ギャップ
【先行指標】KPIに先行して動く指標を特定
【測定方法】データソース×取得頻度×集計ロジック
【アラート基準】異常値の定義と対応フロー
```

## 出力フォーマット
```
【結論】最重要KPIとその理由
【現状分析】データに基づく現状（グラフ/表で可視化）
【課題特定】ボトルネックの場所と影響度
【改善施策】施策×想定改善幅×PLインパクト
【測定計画】いつ何をどう測るか
```
